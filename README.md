# LogoFit

形がバラバラな企業ロゴ（SVG / PNG）を、**見た目の大きさ（塗り面積）を揃えて**指定サイズの枠に配置し、複数サイズの PNG を一括書き出しするブラウザツール。

イベントのスポンサー一覧ページ、スポンサーボード、パートナー/導入企業一覧などの素材づくりを想定。例: [RubyKaigi のスポンサーページ](https://rubykaigi.org/2026/sponsors/)。

完全クライアントサイド動作で、**ファイルは外部送信しません**。作業中のファイルと設定は、再読み込みから復旧できるようブラウザ内の IndexedDB に自動保存します。

---

## 主な機能

- SVG / PNG の複数アップロード（ドラッグ&ドロップ / ファイル選択）
- 透明余白の自動除去（実描画範囲の検出）
- **塗り面積ベースの視覚サイズ補正**（バウンディングボックスではなく実際の塗り面積で揃える）
- 出力プリセットを任意個数（prefix / 幅 / 高さ）で管理
- ロゴごとの面積率の個別調整
- カード表示での書き出しファイル名編集（拡張子は入力形式に応じて固定表示）
- 書き出し背景（透明 / 白）の切り替え
- 書き出し設定へ影響しないプレビュー用白背景の切り替え
- 「カード」表示と「並べて」表示（書き出し前のバランス確認）
- アップロード元ファイル、設定、個別調整のブラウザ内自動保存
- 複数ファイル解析時の進捗表示
- 全ロゴ × 全プリセットを ZIP で一括ダウンロード

---

## 補正の仕組み

ロゴをラスタライズ後、非透明ピクセルから**実描画範囲**と **alpha 加重インク面積**を求め、各プリセット枠に対して次のスケールで配置します。

```
基準は「余白の内側（使える領域）」。

f_target = 目標面積率を満たすスケール
f_min    = 最小面積率を満たすスケール（下限）
f_margin = 余白の内側に収まる上限スケール

f = clamp(f_target, f_min, f_margin)   // 余白上限を常に優先
```

- **目標ロゴ面積率** … 全ロゴを揃えたい狙い。形が違っても同じ面積率を狙うので見た目が揃う
- **最小面積率** … 細長いロゴが小さくなりすぎないための下限
- **余白** … 上下 / 左右を個別指定。ロゴはこの内側まで拡大（＝実質の最大）。余白に達したら頭打ち

完璧な面積一致は原理的に不可能なため、制約の範囲内でのベストエフォート。達成できなかった場合はプレビューに状態（`最小` / `余白上限`）を表示します。

### 設定（初期値）

| 項目 | 初期値 | 説明 |
| --- | --- | --- |
| 目標ロゴ面積率 | 25% | 余白の内側に対する塗りの割合 |
| 最小面積率 | 18% | 内側面積に対する塗りの下限 |
| 余白（上下 / 左右） | 10% / 10% | 枠端からの余白（＝最大の上限を規定）。初期状態では縦横を同期 |
| 背景 | 透明 | 書き出す画像へ適用。透明 / 白 |

### 出力プリセット（初期値）

RubyKaigi のスポンサー画像（@2x アセット）の実寸に合わせた正方形。

| Prefix | サイズ |
| --- | --- |
| `ruby_` | 600×600 |
| `plat_` | 500×500 |
| `gold_` | 400×400 |
| `silver_` | 320×320 |

`cookpad.svg` を入力すると `ruby_cookpad.png` `plat_cookpad.png` … が生成されます。

出力サイズは、ブラウザのメモリ枯渇を避けるため最大辺 `4096px`、最大 `16,000,000px` に制限します。入力値は整数へ丸められ、上限を超える場合は安全な範囲へ調整されます。

## 使い方

1. SVG / PNG をアップロードします。
2. 出力プリセットの prefix・幅・高さを調整します。
3. 補正設定で目標ロゴ面積率、最小面積率、余白、書き出し背景を指定します。
4. カード表示でトリム後の元画像、補正後プレビュー、達成面積率を確認します。
5. 必要に応じてカード上部で出力用のベース名を編集し、ロゴごとの面積率を調整します。
6. 「並べて」表示で全体のバランスを確認し、ZIPを生成します。

余白は `上下` と `左右` を個別に指定できます。「縦横を同じにする」を有効にすると、どちらのスライダーを動かしても同じ値へ同期します。

プレビュー操作列の「白背景で確認」は、カード表示と「並べて」表示だけに適用されます。ZIP内のPNG背景は変更しません。書き出し背景を変更する場合は、補正設定の「背景」を使用します。

カード上部のファイル名入力欄では拡張子を除いたベース名を編集します。入力元の `.svg` / `.png` は固定表示され、ZIP内の生成画像は常に `.png` になります。

アップロード元ファイル、設定、プリセット、ファイル名編集、ロゴごとの面積率は IndexedDB に自動保存されます。ページを再読み込みすると、保存済みファイルを再解析して作業状態を復旧します。「全消去」は確認後にロゴと個別調整を削除します。

画面上部には `下書き保存済み` / `下書きを保存中` / `下書き保存失敗` を表示します。ブラウザの保存容量などが原因で保存できなかった場合は、対処方法を含む警告バナーを表示します。不要なロゴを削除するなどして次回の保存に成功すると、保存失敗の警告は解除されます。

復旧に失敗した場合は、既存の下書きを保護するため自動保存を停止し、専用の警告バナーを表示します。利用者が明示的に編集すると、現在の画面内容で自動保存を再開します。

プレビューの操作バーはスクロール中も画面上部に固定されます。ロゴが多い場合でも、表示切り替え、白背景確認、全消去、ZIP生成へ移動せずアクセスできます。

ZIP生成中は、アップロード、設定、プリセット、ファイル名、個別面積率、削除をロックします。生成対象と画面上の設定がずれないようにするためです。

プリセットの幅・高さは入力中に空欄へできます。フォーカスを外すか Enter を押すと、1px以上の値として確定します。

## 使い方（開発）

```bash
npm install
npm run dev      # 開発サーバー（http://localhost:5173/）
npm run build    # 型チェック + 本番ビルド（dist/）
npm run test     # 主要ロジックの回帰テスト
npm run preview  # ビルド成果物のプレビュー
```

`samples/` に動作確認用のロゴがあります。

- `samples/realistic/` … スポンサー一覧に近い見た目の架空ブランド。横長 / 縦組み / 正方形寄り / 縦長を含みます。
- `samples/validation/` … ロジック検証用。`t*.svg` は形状別の回帰確認、`malicious-test.svg` はサニタイズ確認、その他は余白除去や縦横比の確認に使用します。

## デプロイ

完全クライアントサイド動作のため、バックエンドは不要。`npm run build` で生成される `dist/` を静的ホスティングへ置くだけで運用でき、サーバ費用はかかりません（画像処理は利用者のブラウザ内で完結）。

本番は **Cloudflare Pages** + GitHub 自動デプロイで運用しています。

- リポジトリ: <https://github.com/machida/LogoFit>
- `main` ブランチへ push すると Cloudflare Pages が自動でビルド・公開します。

### Cloudflare Pages の設定

| 項目 | 値 |
| --- | --- |
| Framework preset | Vite |
| Build command | `npm run build` |
| Build output directory | `dist` |
| Production branch | `main` |

- ルーティング（react-router 等）を使わない単一ページのため、`_redirects` や Vite の `base` 設定は不要です。
- ビルドが Node のバージョン差異で失敗する場合は、Pages の環境変数に `NODE_VERSION=22` を設定します。
- 独自ドメインは Pages プロジェクトの **Custom domains** から追加できます（SSL は自動・無料）。

## プロジェクト構成

```
src/
  core/                ロジック（フレームワーク非依存）
    types.ts           型定義・デフォルト設定・プリセット
    sanitizeSvg.ts     SVG サニタイズ + 固有サイズ解決
    rasterize.ts       SVG / PNG → ソースキャンバス
    analyze.ts         透明余白トリム + alpha 加重インク面積
    draft.ts           IndexedDB への作業状態保存・復旧
    margin.ts          上下 / 左右余白から利用可能領域を算出
    fit.ts             視覚サイズ補正（面積率→制約クランプ）
    compose.ts         枠への合成・PNG 化・ファイル名
    zip.ts             ZIP まとめ + ダウンロード
    loadLogo.ts        File → 解析済み LogoItem
  components/          UI（Uploader / PresetTable / SettingsPanel / LogoPreview / BoardView）
  App.tsx              全体の状態管理と結線
samples/
  realistic/           実利用に近い見た目の架空ブランド
  validation/          形状・余白除去・サニタイズの検証用ロゴ
```

技術スタック: Vite + React + TypeScript / JSZip。Canvas API・File API・Blob API を利用。

## セキュリティ

SVG は DOM へ直接挿入せず、`DOMParser` で解析して以下を除去してからラスタライズします。

- `<script>` などの実行可能タグ
- `<style>` 要素
- `on*` イベントハンドラ属性
- 外部参照（`href` / `xlink:href` の http / javascript / file / 相対URLなど。内部参照と `data:image` のみ許可）
- 属性値の外部 `url(...)` 参照（`url(#...)` の内部参照のみ許可）

---

## 既知の制限

- **不透明背景の PNG は未対応**（後回し）。現状は全ピクセルが塗り判定となり、余白トリムされず画像全体がロゴ扱いになります。透明 PNG を推奨。

## 対象外（MVP）

印刷用途（CMYK / ICC / PDF/X）、PDF・AI 入力、SVG・PDF 出力は対象外。
